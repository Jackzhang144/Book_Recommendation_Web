Project 1 评分细则与任务

(1) 数据集信息提取（20%）
需要完成（可以理解为基础 EDA）：
1.	显示数据集的 shape（行数、列数）。
2.	统计整个数据集中 缺失值总数。
3.	检测 哪些列存在缺失值。
4.	创建可比较用的 对比数据集（comparison dataset）（文中没细说，可理解为你清洗前/后 或 不同过滤条件的子集）。
5.	显示数据集的 info（类似 df.info()）。
6.	显示每一列的 唯一值数量。
7.	显示各列的 数据类型。

(2) 数据探索与预处理（20%）
•	分析数据特征和分布，检查缺失值。
•	根据特征类型进行处理，包括但不限于：
•	缺失值填补或删除
•	特征编码（如类别变量 One-Hot 编码等）
•	标准化或归一化
•	对你感兴趣的特征做可视化（比如评分分布、用户年龄分布、热门书籍等）。

(3) 算法定义（30%）
•	指定你要使用的推荐算法。
•	要求至少 3 个：
1.	课堂讲过的算法 ≥ 1 个
2.	LightGBM（必须）
3.	参考算法中至少选 1 个复现（LightFM / DCN v2 / DeepFM / BPR / DIN）

(4) 图书推荐实现（20%）
•	根据你定义的各种推荐算法，实现图书推荐功能：
•	输入一本书名，输出 5 本相似书的推荐。
•	可以考虑不同算法的结果对比。

(5) 系统展示（10%）
•	将你的图书推荐系统做成 Web 或移动应用 的形式进行展示。
•	可以是简单的网页（例如用 Flask / Django / Streamlit / Gradio 等），
•	或一个原型级别的移动 App 界面。

最后，给你一点前端实现建议（顺手说一句）

你作为前端，可以按这几个页面来设计：
1.	首页：搜索 + 热门推荐
•	搜索框：调用 GET /books/search
•	搜索结果列表，点击进入书籍详情页
2.	书籍详情页：
•	调用 GET /books/{book_id} 显示图书信息
•	下方调 GET /recommendations/by-book 展示“相似图书”卡片
•	想加分的话，再加一个“算法切换”，调用 /recommendations/by-book-and-algorithm
3.	搜索直接推荐页（简单满足课程要求）：
•	顶上一个输入框（只要书名，不严格选 book_id）
•	按钮调用 GET /recommendations/by-title?q=xxx&k=5
•	显示选中的那本书 + 下面 5 本推荐书
